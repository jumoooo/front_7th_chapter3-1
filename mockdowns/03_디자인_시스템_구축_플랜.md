### Tailwind 독립 구축 Quick Plan (주니어용)

1. **프로젝트 초기화**
   - `pnpm create vite my-ds --template react-ts`
   - `cd my-ds && pnpm install`
   - `pnpm add -D tailwindcss@next @tailwindcss/vite @tailwindcss/postcss`
2. **Tailwind 연결**
   - `tailwind.config.ts` 생성 후 `plugins: [tailwindcss()]`, `content: ["./index.html", "./src/**/*.{ts,tsx}"]`
   - `src/main.tsx`에서 `import 'tailwindcss/tailwind.css'`
   - `src/styles/tokens.css` 파일 만들어 global 변수/레이어 정의 후 main에 추가
3. **디자인 토큰 작성**
   - `:root`에 `--color-primary`, `--radius-md`, `--font-sans` 등 의미 기반 변수 선언
   - `@layer base`에서 body, heading 기본 스타일 지정
   - `tailwind.config.ts` `theme.extend`로 colors, spacing, fontFamily에 CSS 변수 매핑
4. **유틸리티/Helper 정의**
   - `@layer utilities`에 `.text-subtle`, `.card-shadow` 등 팀 공통 유틸 추가
   - 상태별 컬러(성공/경고)는 `theme.extend.colors.status` 형태로 묶어서 관리
5. **컴포넌트 패턴 구축**
   - `pnpm add class-variance-authority` 후 `src/lib/cva.ts`에 공통 helper 작성
   - Button → Badge 순으로 `cva`로 variant/size 설계
   - shadcn/ui를 가져와야 한다면 `npx shadcn-ui@latest init` 후 필요한 프리미티브 복사
6. **검증 루틴**
   - `pnpm dev`로 시각 확인, `pnpm lint`로 코드 규칙 체크
   - Storybook 도입 시 `pnpm dlx storybook@latest init` → 컴포넌트별 스토리로 API 검증

> 😊 **팁:** 토큰 → 유틸 → 컴포넌트 순으로 반복하면 규모가 커져도 구조가 무너지지 않는다. 필요한 것만 작게 도출하고 바로 스토리/페이지에서 검증한다.
