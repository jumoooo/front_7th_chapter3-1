# 🌙 다크 모드 현재 상태 분석

## 현재 다크 모드가 적용되는 부분

### ✅ 작동하는 부분

1. **CSS 변수 정의**
   - `globals.css`에 `.dark` 클래스가 정의되어 있음
   - 다크 모드 토큰이 모두 정의되어 있음

2. **테마 토글 기능**
   - `useTheme` hook이 `dark` 클래스를 `document.documentElement`에 추가/제거
   - 테마 변경 버튼이 정상 작동

3. **일부 컴포넌트**
   - `Card` 컴포넌트: `bg-card` 사용 → 다크 모드에서 어두운 색으로 변경됨
   - `Button` 컴포넌트: 일부 variant에서 `dark:` 접두사 사용
   - `Input`, `Select`, `Textarea`: `dark:` 접두사 사용

### ❌ 작동하지 않는 부분

1. **ManagementPage 배경색**
   - `bg-[#f0f0f0]` 하드코딩 → 다크 모드에서도 밝은 회색 유지
   - 해결: `bg-background` 사용 필요

2. **섹션 배경색**
   - `bg-white` 하드코딩 → 다크 모드에서도 흰색 유지
   - 해결: `bg-card` 사용 필요

3. **테두리 색상**
   - `border-[#ddd]` 하드코딩 → 다크 모드에서도 밝은 회색 유지
   - 해결: `border-border` 사용 필요

4. **텍스트 색상**
   - `text-[#333]`, `text-[#666]` 하드코딩 → 다크 모드에서도 어두운 색 유지
   - 해결: `text-foreground`, `text-muted-foreground` 사용 필요

5. **통계 카드**
   - 인라인 스타일로 CSS 변수 직접 사용 → 다크 모드에서도 동일한 색상 유지
   - 해결: 다크 모드용 색상 변수 추가 필요

## 다크 모드에서 변경되는 CSS 변수

### 라이트 모드 → 다크 모드

| 변수 | 라이트 모드 | 다크 모드 |
|------|------------|----------|
| `--color-background` | `#f0f0f0` (neutral-200) | `#1a1a1a` (neutral-900) |
| `--color-foreground` | `#333` (neutral-700) | `#fff` (neutral-50) |
| `--color-card` | `#fff` (neutral-50) | `#2a2a2a` (neutral-800) |
| `--color-card-foreground` | `#333` (neutral-700) | `#fff` (neutral-50) |
| `--color-border` | `#ddd` (neutral-300) | `#4a4a4a` (neutral-700) |
| `--color-primary` | `#1976d2` (primary-500) | `#60a5fa` (primary-400) |
| `--color-destructive` | `#d32f2f` (error-500) | `#f87171` (error-400) |

## 해결 방법

1. **하드코딩된 색상 제거**
   - `bg-[#f0f0f0]` → `bg-background`
   - `bg-white` → `bg-card`
   - `border-[#ddd]` → `border-border`
   - `text-[#333]` → `text-foreground`
   - `text-[#666]` → `text-muted-foreground`

2. **다크 모드 대응 추가**
   - 통계 카드 색상도 다크 모드에서 조정
   - 테이블, 모달 등 모든 컴포넌트에 다크 모드 스타일 적용

3. **테스트**
   - 다크 모드 토글 버튼 클릭 시 모든 요소가 올바르게 변경되는지 확인
